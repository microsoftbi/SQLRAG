# 2026-06-10 改动汇总

## 1. API_Graph.py — LLM 调用改为 langchain_deepseek

`call_llm_generate_sql` 函数：
- 导入从 `from openai import OpenAI` 改为 `from langchain_deepseek import ChatDeepSeek` + `from langchain_core.messages import HumanMessage`
- 客户端初始化从 `OpenAI(api_key=..., base_url=...)` 改为 `ChatDeepSeek(model=..., api_key=..., base_url=..., temperature=0, max_tokens=2000)`
- 请求从 `client.chat.completions.create(messages=[{...}])` 改为 `llm.invoke([HumanMessage(content=prompt)])`
- 结果提取从 `response.choices[0].message.content` 改为 `response.content`
- max_tokens 从 2000 提升到 5000

**Why:** 统一项目中 LLM 调用方式为 LangChain 接口，便于后续切换模型和链式调用。
**How to apply:** 后续新 LLM 调用应优先使用 `ChatDeepSeek` + `invoke()` 模式。

## 2. 提示词抽离到独立 md 文件

新增 `backend/prompts/` 目录及其下 4 个文件：

| 文件 | 对应函数 |
|------|---------|
| `backend/prompts/graph_sql_generation.md` | `API_Graph.py::call_llm_generate_sql` — 图查询 SQL 生成 |
| `backend/prompts/graph_sql_fix.md` | `API_Graph.py::call_llm_fix_sql` — SQL 出错修正 |
| `backend/prompts/vector_semantic_chunk.md` | `API_Vector.py::semantic_chunk_document` — 语义分块 |
| `backend/prompts/vector_qa_answer.md` | `API_Vector.py::generate_answer_with_sources` — 向量 QA 回答 |

每个 md 文件：
- `API_Vector.py` 的 `semantic_chunk_document` 使用 `## System Prompt` / `## User Prompt` 分区，Python 端解析返回元组 `(system_prompt, user_prompt)`
- 其余使用 `{placeholder}` 占位符，Python 端用 `.replace()` 注入

**Why:** 修改提示词不再需要改 Python 代码，产品/运营人员可直接编辑 md 文件。
**How to apply:** 新增 LLM 功能时，prompt 模板一律放在 `backend/prompts/` 下。

## 3. GraphDB SQL 执行出错自动修正

- 新增 `API_Graph.py::call_llm_fix_sql(question, failed_sql, error, schema)` — 将出错信息送回 LLM 修正
- 新增 `API_Graph.py::get_fix_prompt_template()` — 读取 fix prompt 模板
- 修改 `graph_qa()` — 加入 retry 循环：`MAX_RETRIES = 2`（初始生成 + 最多 2 次修正）
- 修正循环使用 `result.startswith("执行SQL时出错")` 判断是否失败

**Why:** SQL 语法错误或列名不对时，让 LLM 自愈而非直接抛给用户。
**How to apply:** 通过修改 MAX_RETRIES 调整重试次数，通过修改 `graph_sql_fix.md` 调整修正提示。

## 4. Legacy GraphDB QA 返回 generated_sql

修改了 `API_Graph.py` 中以下函数，全部改为返回 `(answer_text, sql_string)` 元组：

- `find_relationship_between_nodes` — 多查询循环，返回 SQL 模式描述
- `_find_related_via_match` — 提取 SQL 为独立变量，返回 `(dict, sql)`
- `find_related_entities` — 解包 + 透传
- `find_common_related_entities` — 解包 + 透传
- `find_path_between_nodes` — 返回 `(text, "")`
- `find_common_entities` — 提取 SQL 为独立变量

`legacy_graph_qa` 所有分支解包后返回 `{"answer": result, "generated_sql": generated_sql}`。

**Why:** 前端 GraphDB QA 页面一直展示 SQL 区域，但非 AI 模式下该字段缺失导致前端无显示。
**How to apply:** 新增 legacy 查询函数时需保持 `(answer, sql)` 返回签名。
