# 2026-06-11 改动汇总

## 1. VectorDB QA 去除关键词回退

`API_Vector.py::vector_qa` 中删除了 `VECTOR_SEARCH` 无结果时的关键词回退逻辑（原来搜索 `d.Title LIKE ? OR d.Source LIKE ?`）。

**Why:** 只做纯向量检索，不混入关键词匹配。如果向量检索无结果，直接返回空给 LLM。
**How to apply:** 后续如需增加回退策略，在 `# 使用LLM基于检索到的内容回答问题` 注释之前插入。

## 2. 嵌入模型切换（多次）

| 顺序 | 模型 | 维度 | 说明 |
|------|------|------|------|
| 之前 | `quentinz/bge-large-zh-v1.5` | 1024 | 稳定，正常使用 |
| 第1次 | `bge-m3:latest` | 1024 | 有 NaN 问题，`failed to encode response: json: unsupported value: NaN` |
| 第2次 | `qwen3-embedding:4b` | 2560 | 需改 `VECTOR(1024)` → `VECTOR(2560)` |
| **最终** | **`qwen3-embedding:0.6b`** | **1024** | **轻量稳定，无需改代码** |

**影响文件:** `backend/.env` (模型名), `backend/API_Vector.py` (VECTOR 类型), `00_VectorDB.sql` (建表DDL), 以及 `search_vector.py`, `diagnostic_vector.py`, `insert_vector_test.py`。

**Why:** 尝试更优的嵌入模型，最终选用 qwen3-embedding:0.6b（600M 参数、1024维、无NaN问题、对中文支持好）。
**注意:** 切换模型后，旧向量与新向量不在同一语义空间，需 DROP INDEX → 重新嵌入 → CREATE INDEX。

## 3. NaN 错误容错增强

`API_Vector.py::get_embedding` 修改重试逻辑：

```python
# 修改前：只对 "input length exceeds" 重试
if "input length exceeds the context length" not in error_message:
    raise

# 修改后：对 NaN 也重试（截断文本）
if "input length exceeds the context length" not in error_message and "NaN" not in error_message:
    raise
```

重试序列：`None → 1200 → 800 → 500 → 300 → 150` 字符，全部失败则跳过该分块。

**Why:** bge-m3 在某些输入上产生 NaN，导致 Ollama 返回 500 错误，重试能绕过问题片段。

## 4. DeepSeek 关闭 thinking

`API_Graph.py::call_llm_generate_sql` 的 ChatDeepSeek 初始化中加入：

```python
extra_body={"thinking": {"type": "disabled"}}
```

仅加了 `call_llm_generate_sql`，`call_llm_fix_sql` 未改动。

**Why:** 避免 DeepSeek 模型输出推理过程，只返回 SQL 语句。
