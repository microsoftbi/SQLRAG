## 1. 项目概述

SQLRAG 是一个基于 SQL Server 2025 原生 VectorDB 和 GraphDB 支持的 RAG (Retrieval-Augmented Generation) 系统，提供知识图谱可视化和向量检索问答功能。

**核心特性：**
- 知识图谱可视化与交互
- 基于 LLM 的图数据库问答
- 文档管理与上传
- 文档自动分块
- 基于向量检索的智能问答

---

## 2. GraphDB 功能

### 2.1 全景视图

**功能描述：**
展示知识图谱的完整视图，支持节点的点击查看和子图探索。

**主要功能：**
- **图谱可视化**：使用 Vis Network 展示所有节点和边
- **节点分类显示**：不同类型的节点使用不同样式
- **点击探索**：点击任意节点弹出子图窗口
- **深度控制**：可调节显示深度（1-5层）
- **动态更新**：调整深度后实时更新子图
- **统计信息**：显示节点数、边数等统计

**操作说明：**
1. 在页面中查看完整的知识图谱
2. 点击任意节点查看详细关联
3. 使用滑块调整显示深度
4. 查看子图的节点和边统计

---

### 2.2 QA 问答

**功能描述：**
基于知识图谱的智能问答系统，支持使用 LLM 自动生成 SQL Server 图查询。

**主要功能：**
- **问题输入**：支持自然语言问题
- **SQL 生成**：可选使用 LLM 生成图查询
- **查询执行**：自动执行 SQL 并格式化结果
- **结果展示**：以易读的表格形式展示答案
- **日志记录**：记录 LLM 交互和生成的 SQL

**使用方式：**
1. 在输入框中输入问题
2. 可选勾选"使用AI生成SQL查询"
3. 点击"提问"按钮
4. 查看生成的 SQL 和查询结果

**示例问题：**
```
方超和周荣是什么关系？
谁认识方超，同时又和周荣有关系？
叶剑遇害案涉及哪些人物？
哪些人物同时出现在多个案件中？
```

**技术实现：**
- 使用 MATCH 子句进行图查询
- 支持 CTE (Common Table Expressions)
- 多关系查询使用 UNION ALL 组合
- 结果使用 STRING_AGG 聚合

## 3. VectorDB 功能

### 3.1 文档管理

**功能描述：**
文档管理界面，支持查看和添加文档。

**主要功能：**
- **文档列表**：显示所有文档的 ID、标题、来源、创建时间
- **添加文档**：弹窗表单，支持手动添加文档
- **查看分块**：点击链接查看文档的所有分块
- **操作列**：每文档提供"查看分块"操作

**添加文档字段：**
- **标题**：文档标题
- **内容**：文档完整内容
- **来源**：文档来源说明

---

### 3.2 文档上传

**功能描述：**
支持多种格式的文档上传，自动提取文本内容。

**主要功能：**
- **拖拽上传**：支持将文件拖拽到上传区域
- **点击上传**：点击区域选择文件
- **格式支持**：
  - `.txt` - 纯文本文件
  - `.md` - Markdown 文件
  - `.docx` - Microsoft Word 文档
  - `.pdf` - PDF 文档
- **进度显示**：显示已上传文件的列表和状态
- **文件大小**：自动格式化显示文件大小
- **状态跟踪**：显示上传成功或失败状态

**上传流程：**
1. 选择或拖拽文件到上传区域
2. 系统自动提取文件内容
3. 保存到数据库并刷新文档列表
4. 显示上传结果和状态

---

### 3.3 分块查看

**功能描述：**
查看文档被自动分块后的所有文本块。

**主要功能：**
- **分块列表**：显示所有分块的详细信息
- **内容展示**：支持滚动查看长内容
- **自动分块**：
  - 每块 500 字符
  - 重叠 100 字符
  - 保留段落格式
- **分块属性**：
  - ChunkId - 分块 ID
  - ChunkIndex - 分块序号
  - ChunkText - 分块内容
  - CreatedAt - 创建时间

**触发条件：**
- 点击文档列表的"查看分块"链接
- 如果文档没有分块，系统会自动创建

---

### 3.4 智能问答

**功能描述：**
基于向量检索的 RAG 问答系统，从文档中检索相关内容并使用 LLM 生成回答。

**主要功能：**
- **问题输入**：自然语言问题输入
- **文档检索**：基于关键词匹配检索相关文档
- **来源展示**：显示参考的文档来源
- **答案生成**：使用 LLM 基于检索内容生成回答
- **格式化展示**：友好的答案和来源展示

**问答流程：**
1. 用户输入问题
2. 系统检索相关文档
3. 构建上下文（最多 5 个文档）
4. 调用 LLM 生成回答
5. 显示答案和参考来源

**来源展示：**
- 文档标题
- 文档内容（截取前 1000 字符）
- 文档来源
- 时间线形式展示

## 4. 后端 API

### 4.1 GraphDB API

#### GET /graph/data
获取知识图谱的所有节点和边数据。

**响应：**
```json
{
  "nodes": [...],
  "edges": [...]
}
```

#### POST /graph/qa
知识图谱问答接口。

**请求：**
```json
{
  "question": "方超和周荣是什么关系？",
  "use_llm": true
}
```

**响应：**
```json
{
  "answer": "查询结果...",
  "generated_sql": "SELECT ... MATCH ..."
}
```

---

### 4.2 VectorDB API

#### GET /vector/documents
获取所有文档列表。

**响应：**
```json
{
  "documents": [
    {
      "DocumentId": 1,
      "Title": "文档标题",
      "Source": "来源",
      "CreatedAt": "2024-01-01T00:00:00"
    }
  ]
}
```

#### POST /vector/documents
手动添加文档。

**请求：**
```json
{
  "title": "文档标题",
  "content": "文档内容",
  "source": "来源"
}
```

#### POST /vector/documents/upload
上传文档文件。

**请求：**
- Content-Type: `multipart/form-data`
- 参数: `file` (文件对象)

**响应：**
```json
{
  "message": "Document uploaded successfully",
  "filename": "example.txt"
}
```

#### GET /vector/documents/{document_id}/chunks
获取指定文档的所有分块。

**响应：**
```json
{
  "chunks": [
    {
      "ChunkId": 1,
      "DocumentId": 1,
      "ChunkIndex": 0,
      "ChunkText": "分块内容",
      "CreatedAt": "2024-01-01T00:00:00"
    }
  ]
}
```

#### POST /vector/qa
向量检索问答接口。

**请求：**
```json
{
  "question": "用户的问题"
}
```

**响应：**
```json
{
  "answer": "生成的回答",
  "sources": [
    {
      "Title": "文档标题",
      "Content": "文档内容",
      "Source": "来源"
    }
  ]
}
```

## 5. 配置说明

### 环境变量配置

在 `backend/.env` 文件中配置：

```env
# 数据库配置
SERVER=LOCALHOST
PORT=1433
DATABASE_GRAPH=GraphDB
DATABASE_VECTOR=VectorDB
USER=sa
PASSWORD=Passw0rd

# 后端端口
BACKEND_PORT=8798

# LLM 配置
LLM_API_KEY=your_api_key_here
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL=deepseek-chat
```

### 日志配置

- **日志文件**：`backend/llm_sql_logs.log`
- **日志内容**：
  - LLM API Key (后 10 位脱敏)
  - LLM Base URL
  - LLM Model
  - 用户问题
  - 生成的 SQL (GraphDB)
  - 生成的回答 (VectorDB)
  - 文档上传记录
  - 分块创建记录

### 分块策略

- **分块大小**：500 字符
- **重叠大小**：100 字符
- **触发条件**：首次查看分块时自动创建
- **存储位置**：`TextChunks` 表

## 附录

### A. 数据库表结构

**GraphDB:**
- `Person` - 人物节点
- `CaseNode` - 案件节点
- `Organization` - 组织节点
- `Location` - 位置节点
- `Item` - 物品节点
- `Event` - 事件节点
- 各类边表（WorksAt, Investigates, SuspectOf 等）

**VectorDB:**
- `Documents` - 文档表
- `TextChunks` - 文本分块表
- `VectorIndex` - 向量索引表（预留）
