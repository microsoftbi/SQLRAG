## 1. 项目概述

SQLRAG 是一个基于 SQL Server 2025 原生 VectorDB 和 GraphDB 支持的 RAG (Retrieval-Augmented Generation) 系统，提供知识图谱可视化和向量检索问答功能。

**核心特性：**
- 知识图谱可视化与交互
- 基于 LLM 的图数据库问答
- 文档管理与上传
- 文档分块（固定分块 / 语义分块）
- 知识库分组管理
- 基于向量检索的智能问答

## 2. GraphDB 功能

### 2.1 全景视图

**功能描述：**
展示知识图谱的完整视图，支持节点的点击查看和子图探索。

**主要功能：**
- **图谱可视化**：使用 Vis Network 展示所有节点和边
- **节点分类显示**：不同类型的节点使用不同样式
- **点击探索**：点击任意节点弹出子图窗口
- **深度控制**：可调节显示深度（1-5层）
- **动态更新**：调整深度后实时更新子图
- **统计信息**：显示节点数、边数等统计

**操作说明：**
1. 在页面中查看完整的知识图谱
2. 点击任意节点查看详细关联
3. 使用滑块调整显示深度
4. 查看子图的节点和边统计

---

### 2.2 QA 问答

**功能描述：**
基于知识图谱的智能问答系统，支持使用 LLM 自动生成 SQL Server 图查询。

**主要功能：**
- **问题输入**：支持自然语言问题
- **SQL 生成**：可选使用 LLM 生成图查询
- **查询执行**：自动执行 SQL 并格式化结果
- **结果展示**：以易读的表格形式展示答案
- **日志记录**：记录 LLM 交互和生成的 SQL

**使用方式：**
1. 在输入框中输入问题
2. 可选勾选"使用AI生成SQL查询"
3. 点击"提问"按钮
4. 查看生成的 SQL 和查询结果

**示例问题：**
```
方超和周荣是什么关系？
谁认识方超，同时又和周荣有关系？
叶剑遇害案涉及哪些人物？
哪些人物同时出现在多个案件中？
```

**技术实现：**
- 使用 MATCH 子句进行图查询
- 支持 CTE (Common Table Expressions)
- 多关系查询使用 UNION ALL 组合
- 结果使用 STRING_AGG 聚合

## 3. VectorDB 功能

### 3.1 文档管理

**功能描述：**
文档管理界面，支持查看、添加、删除文档，以及修改文档所属知识库。

**主要功能：**
- **文档列表**：显示所有文档的 ID、标题、所属知识库、来源、创建时间
- **添加文档**：弹窗表单，支持手动添加文档并选择所属知识库
- **删除文档**：点击"删除"按钮，级联软删除文档及其所有分块和向量数据
- **修改知识库**：点击文档所属知识库名称，行内直接切换知识库
- **知识库筛选**：顶部的下拉框可按知识库筛选文档列表
- **查看分块**：点击链接查看文档的所有分块

**添加文档字段：**
- **标题**：文档标题
- **内容**：文档完整内容
- **来源**：文档来源说明
- **知识库**：选择所属知识库（可选）

---

### 3.2 文档上传与分块

**功能描述：**
支持多种格式的文档上传，上传后文档暂不入库，用户可选择分块方式并确认后再正式入库。

**主要功能：**
- **拖拽上传**：支持将文件拖拽到上传区域
- **点击上传**：点击区域选择文件
- **格式支持**：
  - `.txt` - 纯文本文件
  - `.md` - Markdown 文件
  - `.docx` - Microsoft Word 文档
  - `.pdf` - PDF 文档
- **上传到知识库**：上传时可指定文档归属的知识库

**上传流程：**
1. 选择或拖拽文件到上传区域
2. 系统自动提取文件内容，仅保存到数据库（不切块、不入库）
3. 在"固定分块"或"语义分块" tab 中选择分块方式
4. 点击"分块预览"查看分块结果（固定分块），或"开始语义分块"（语义分块）
5. 确认分块结果后，点击"入库"将分块和向量数据写入数据库

---

### 3.3 固定分块

**功能描述：**
使用 LangChain RecursiveCharacterTextSplitter 按固定字符数进行文本分块。

**主要功能：**
- **Chunk Size**：设置每个分块的最大字符数（默认从系统配置读取，可临时改写）
- **Chunk Overlap**：设置相邻分块之间的重叠字符数（默认从系统配置读取，可临时改写）
- **分块预览**：点击"分块预览"查看分块结果（不入库）
- **入库**：确认分块结果后，点击"入库"将分块和向量数据写入数据库
- **已上传文档列表**：显示所有已上传但尚未入库的文档，提供分块预览和入库操作

---

### 3.4 语义分块

**功能描述：**
调用配置的大模型对文档进行智能语义分段。

**主要功能：**
- **选择文档**：从已上传的文档列表中选择一个文档
- **开始语义分块**：调用 LLM 对文档内容进行语义分析，自动分段
- **原文/结果对比**：左侧显示原文，右侧显示分块结果（每块带编号和长度）
- **文档入库**：确认分块结果后，点击"文档入库"将语义分块结果写入数据库并生成向量嵌入

---

### 3.5 知识库管理

**功能描述：**
对知识库进行增删改查管理，用于对文档进行分组。

**主要功能：**
- **知识库列表**：显示所有知识库的 ID、名称、描述、创建时间
- **创建知识库**：弹窗表单，输入名称和描述
- **编辑知识库**：修改知识库名称和描述
- **删除知识库**：软删除知识库，同时将下属文档的 KnowledgeBaseId 置为 NULL

---

### 3.6 系统配置

**功能描述：**
在线查看和修改系统运行参数。

**可配置参数：**
| 参数 | 说明 | 默认值 |
|------|------|--------|
| Chunk Size | 每个文本块的最大字符数 | 1000 |
| Chunk Overlap | 相邻文本块之间的重叠字符数 | 200 |
| Embedding Model | Ollama 上的文本嵌入模型名称 | nomic-embed-text |
| Ollama Base URL | Ollama 服务地址 | http://localhost:11434 |

**操作方式：**
1. 修改参数值
2. 点击"保存配置"
3. 配置即时写入 `.env` 文件并同步到内存，无需重启后端

---

### 3.7 智能问答

**功能描述：**
基于向量检索的 RAG 问答系统，从文档中检索相关内容并使用 LLM 生成回答。

**主要功能：**
- **问题输入**：自然语言问题输入
- **文档检索**：基于关键词匹配检索相关文档
- **来源展示**：显示参考的文档来源
- **答案生成**：使用 LLM 基于检索内容生成回答
- **格式化展示**：友好的答案和来源展示

**问答流程：**
1. 用户输入问题
2. 系统检索相关文档
3. 构建上下文（最多 5 个文档）
4. 调用 LLM 生成回答
5. 显示答案和参考来源

**来源展示：**
- 文档标题
- 文档内容（截取前 1000 字符）
- 文档来源
- 时间线形式展示

## 4. 后端 API

### 4.1 GraphDB API

#### GET /graph/data
获取知识图谱的所有节点和边数据。

**响应：**
```json
{
  "nodes": [...],
  "edges": [...]
}
```

#### POST /graph/qa
知识图谱问答接口。

**请求：**
```json
{
  "question": "方超和周荣是什么关系？",
  "use_llm": true
}
```

**响应：**
```json
{
  "answer": "查询结果...",
  "generated_sql": "SELECT ... MATCH ..."
}
```

---

### 4.2 VectorDB API

#### GET /vector/documents
获取所有文档列表，支持按知识库筛选。

**请求参数：**
- `knowledge_base_id` (可选) — 按知识库筛选

**响应：**
```json
{
  "documents": [
    {
      "DocumentId": 1,
      "Title": "文档标题",
      "KnowledgeBaseId": 1,
      "KnowledgeBaseName": "知识库名称",
      "Source": "来源",
      "CreatedAt": "2024-01-01T00:00:00"
    }
  ]
}
```

#### POST /vector/documents
手动添加文档。

**请求：**
```json
{
  "title": "文档标题",
  "content": "文档内容",
  "source": "来源",
  "knowledge_base_id": 1
}
```

#### PUT /vector/documents/{document_id}
更新文档信息（支持修改所属知识库）。

**请求：**
```json
{
  "knowledge_base_id": 2
}
```

#### DELETE /vector/documents/{document_id}
软删除文档及其关联的 TextChunks 和 VectorIndex 记录。

**响应：**
```json
{
  "message": "文档及关联数据已删除"
}
```

#### POST /vector/documents/upload
上传文档文件（仅保存，不自动切块和嵌入）。

**请求：**
- Content-Type: `multipart/form-data`
- 参数: `file` (文件对象), `knowledge_base_id` (可选)

**响应：**
```json
{
  "message": "Document uploaded successfully",
  "filename": "example.txt",
  "documentId": 1
}
```

#### GET /vector/documents/{document_id}/chunks
获取指定文档的所有分块。

**响应：**
```json
{
  "chunks": [
    {
      "ChunkId": 1,
      "DocumentId": 1,
      "ChunkIndex": 0,
      "ChunkText": "分块内容",
      "CreatedAt": "2024-01-01T00:00:00"
    }
  ]
}
```

#### POST /vector/documents/{document_id}/preview-chunks
预览固定分块结果（不入库）。

**请求：**
```json
{
  "chunk_size": 1000,
  "chunk_overlap": 200
}
```

**响应：**
```json
{
  "chunks": [
    {"ChunkIndex": 0, "ChunkText": "...", "Length": 500}
  ],
  "total_chunks": 5
}
```

#### POST /vector/documents/{document_id}/semantic-chunk
调用大模型对文档进行语义分块。

**响应：**
```json
{
  "original_content": "文章原文...",
  "chunks": [
    {"ChunkIndex": 0, "ChunkText": "...", "Length": 500}
  ],
  "total_chunks": 5
}
```

#### POST /vector/documents/{document_id}/commit-chunks
按指定参数进行固定分块并入库，同时生成向量嵌入。

**请求：**
```json
{
  "chunk_size": 1000,
  "chunk_overlap": 200
}
```

#### POST /vector/documents/{document_id}/commit-chunks-raw
将指定的分块列表直接入库（用于语义分块结果），并生成向量嵌入。

**请求：**
```json
{
  "chunks": ["第一段...", "第二段...", "..."]
}
```

#### POST /vector/documents/{document_id}/embed
为指定文档的分块生成嵌入向量。

#### GET /vector/config
获取系统配置。

**响应：**
```json
{
  "chunk_size": 1000,
  "chunk_overlap": 200,
  "embedding_model": "nomic-embed-text",
  "ollama_base_url": "http://localhost:11434"
}
```

#### PUT /vector/config
更新系统配置（写入 .env 文件并同步到内存）。

**请求：**
```json
{
  "chunk_size": 500,
  "chunk_overlap": 100,
  "embedding_model": "nomic-embed-text",
  "ollama_base_url": "http://localhost:11434"
}
```

#### GET /vector/knowledge-bases
获取所有知识库列表。

**响应：**
```json
{
  "knowledge_bases": [
    {
      "KnowledgeBaseId": 1,
      "Name": "知识库名称",
      "Description": "描述",
      "CreatedAt": "2024-01-01T00:00:00"
    }
  ]
}
```

#### POST /vector/knowledge-bases
创建知识库。

**请求：**
```json
{
  "name": "知识库名称",
  "description": "描述"
}
```

#### PUT /vector/knowledge-bases/{kb_id}
更新知识库名称/描述。

#### DELETE /vector/knowledge-bases/{kb_id}
软删除知识库，同时将下属文档的 KnowledgeBaseId 置 NULL。

#### POST /vector/qa
向量检索问答接口。

**请求：**
```json
{
  "question": "用户的问题"
}
```

**响应：**
```json
{
  "answer": "生成的回答",
  "sources": [
    {
      "Title": "文档标题",
      "Content": "文档内容",
      "Source": "来源"
    }
  ]
}
```

## 5. 配置说明

### 环境变量配置

在 `backend/.env` 文件中配置：

```env
# 数据库配置
SERVER=LOCALHOST
PORT=1433
DATABASE_GRAPH=GraphDB
DATABASE_VECTOR=VectorDB
USER=sa
PASSWORD=Passw0rd

# 后端端口
BACKEND_PORT=8798

# 分块配置
CHUNK_SIZE=1000
CHUNK_OVERLAP=200

# Embedding 配置
EMBEDDING_MODEL=nomic-embed-text
OLLAMA_BASE_URL=http://localhost:11434

# LLM 配置
LLM_API_KEY=your_api_key_here
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL=deepseek-chat
```

### 日志配置

- **日志文件**：`backend/llm_sql_logs.log`
- **日志内容**：
  - LLM API Key (后 10 位脱敏)
  - LLM Base URL
  - LLM Model
  - 用户问题
  - 生成的 SQL (GraphDB)
  - 生成的回答 (VectorDB)
  - 文档上传记录
  - 分块创建记录
  - 知识库操作记录
  - 配置更新记录

### 分块策略

| 方式 | 说明 | 参数来源 |
|------|------|----------|
| 固定分块 | 使用 LangChain RecursiveCharacterTextSplitter 按字符数切分 | `.env` 中的 CHUNK_SIZE / CHUNK_OVERLAP，上传时可临时改写 |
| 语义分块 | 调用配置的大模型对文档进行智能语义分段 | 需配置 LLM_API_KEY |

## 附录

### A. 数据库表结构

**GraphDB:**
- `Person` - 人物节点
- `CaseNode` - 案件节点
- `Organization` - 组织节点
- `Location` - 位置节点
- `Item` - 物品节点
- `Event` - 事件节点
- 各类边表（WorksAt, Investigates, SuspectOf 等）

**VectorDB:**
- `KnowledgeBases` - 知识库表
- `Documents` - 文档表（关联 KnowledgeBases）
- `TextChunks` - 文本分块表（关联 Documents）
- `VectorIndex` - 向量索引表（关联 TextChunks，含 VECTOR INDEX）
